# 📝 文章式報告說明

> 文章式自動生成報告已移至 `market-intelligence-system`。本文件保留原流程的參考說明。

## 📋 概覽

從 2025-12-01 開始，`make daily` 命令生成的報告已優化為**結構一致**且**自然流暢**的文章式風格。

### 核心改進

✅ **結構一致性** - 每次生成的報告遵循相同的章節架構
✅ **文章風格** - 自然流暢的專業敘述，而非冷冰冰的數據堆砌
✅ **深度分析** - 提供洞見和判斷，而非僅僅轉述數據
✅ **易於閱讀** - 數據用表格呈現，分析融入文字敘述

---

## 🎯 與舊版的區別

### 舊版問題
- ❌ 每次生成的格式可能不一致
- ❌ 章節順序可能變化
- ❌ 分析深度不穩定
- ❌ 有時過於條列式，缺乏連貫性

### 新版優勢
- ✅ 使用標準模板，確保結構一致
- ✅ 所有章節按固定順序排列
- ✅ 明確的分析深度要求
- ✅ 強調文章式敘述，保持專業性

## 📊 報告結構

每份報告包含以下標準章節：

### 1. 📊 執行摘要
- 市場綜述 (2-3段文字)
- 關鍵數據表格
- 風險等級評估

### 2. 🌍 全球市場總覽
- 美國市場 (主要指數表格 + 分析)
- 亞洲市場 (8個市場 + 分析)
- 歐洲市場 (4個市場 + 分析)
- 市場驅動因素總結

### 3. 💼 持倉股票分析
- 整體持倉表現
- 重大虧損股票 (>2% 跌幅) - 深度分析
- 輕微下跌股票 (0-2% 跌幅) - 簡要分析
- 強勢上漲股票 (>2% 漲幅) - 深度分析
- 穩健表現股票 (0-2% 漲幅) - 簡要分析
- 持倉操作建議總結

### 4. 📰 重要新聞解讀
- 按產業分類 (科技、醫療、金融等)
- 深度分析，而非轉述
- 解釋市場影響和投資啟示

### 5. ⚠️ 風險評估與投資機會
- VIX 恐慌指數分析
- 商品市場 (黃金、原油、比特幣)
- 債券市場
- 風險矩陣

### 6. 💡 投資策略建議
- 短期策略 (1-2週) - 具體可執行
- 中期策略 (1-3個月) - 配置調整
- 長期策略 (6-12個月) - 核心主題

### 7. 🔮 後市展望
- 關鍵催化劑
- 情境分析 (樂觀/基準/悲觀)
- 未來展望總結

### 8. 📋 關鍵監控指標
- 每日監控指標
- 每週監控事項
- 持倉股票監控重點

### 9. ✅ 行動清單
- 立即執行 (本週內)
- 中期規劃 (月中前)
- 長期追蹤

## 🎨 寫作風格特色

### 強調的風格
- ✅ 完整的句子和段落
- ✅ 邏輯連貫的敘述
- ✅ 數據服務於分析
- ✅ 提供「為什麼」的解釋
- ✅ 專業但易懂

### 避免的風格
- ❌ 機械式列點
- ❌ 冷冰冰的條列結構
- ❌ 單純羅列數據
- ❌ 過於學術或過於口語
- ❌ 像填表格一樣的內容

## 📚 使用方式

### 生成報告

```bash
# 標準方式 (推薦)
make daily

# 這會執行:
# 1. 爬取市場數據
# 2. 爬取持倉股票價格和新聞
# 3. 使用模板生成文章式報告
```

### 查看報告

```bash
# 查看今天的報告
cat analysis/market/$(date +%Y-%m-%d).md

# 或用編輯器打開
code analysis/market/$(date +%Y-%m-%d).md
```

## 🔧 模板系統

### 模板位置
- **報告模板**: `templates/analysis/market-daily-article-template.md`
- **生成腳本**: `tools/utils/analyze_daily_market.sh`

### 工作原理

1. 腳本讀取報告模板
2. 腳本讀取最近一份報告作為範例
3. 組合成 Claude 提示詞，包含：
   - 結構化要求 (來自模板)
   - 文章風格要求
   - 範例參考 (最近的報告)
   - 今日市場數據
4. Claude 生成遵循模板的文章式報告

### 自訂模板

如果需要調整報告結構，可以編輯模板文件：

```bash
# 編輯模板
code templates/analysis/market-daily-article-template.md

# 下次執行 make daily 時會使用新模板
```

## 📊 與 YAML 格式的關係

系統現在提供兩種格式：

### 文章式 Markdown (預設)
```bash
make daily
```
- 適合：人類閱讀、分享報告
- 特色：自然流暢、專業分析
- 輸出：`analysis/market/YYYY-MM-DD.md`

### YAML 格式 (進階)
```bash
make daily-yaml
```
- 適合：程式處理、數據分析
- 特色：結構化數據、易於解析
- 輸出：`analysis/market/YYYY-MM-DD.yaml`

兩種格式可以並存使用，各取所需。

## 💡 最佳實踐

### 1. 日常使用
- 每天執行 `make daily` 生成報告
- 閱讀報告，關注重點章節
- 根據行動清單執行操作

### 2. 報告品質
- 第一份報告可能需要手動調整
- 調整後的報告會成為新範例
- 後續報告會參考這份範例
- 逐步提升報告品質

### 3. 反饋改進
- 如果報告格式不符預期，可以：
  - 編輯模板文件
  - 調整生成腳本的提示詞
  - 手動修正一份報告作為新範例

## ❓ 常見問題

### Q1: 為什麼報告格式還是不一致？

**A**: 可能的原因:
1. 模板文件不存在或路徑錯誤
2. Claude 沒有完全遵循模板
3. 提示詞需要調整

**解決方案**:
```bash
# 檢查模板是否存在
ls -l templates/analysis/market-daily-article-template.md

# 如果不存在，重新創建
# (參考本文檔的模板系統部分)
```

### Q2: 報告太冷冰冰，不像文章？

**A**: 確保腳本包含文章風格要求：

檢查 `tools/utils/analyze_daily_market.sh` 中的提示詞是否包含：
- "文章式風格"要求
- "避免機械式列點"指示
- "用完整句子和段落"說明

### Q3: 想回到舊版格式？

**A**: 可以暫時重命名模板：
```bash
mv templates/analysis/market-daily-article-template.md \
   templates/analysis/market-daily-article-template.md.backup
```

下次執行 `make daily` 時會使用預設格式。

### Q4: 想用 YAML 格式？

**A**: 使用 YAML 命令：
```bash
make daily-yaml
```

詳細說明參考 [YAML-WORKFLOW.md](YAML-WORKFLOW.md)
